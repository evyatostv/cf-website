>    Never publish ₪759. If the owner reconciles to ₪759, fix this kit first.

## 4. Screenshots checklist

Directories rank listings with images higher. Capture **6–10** clean screenshots.

**Rules for every screenshot:**
- [ ] Use **demo/fake patient data only** — no real names, IDs, phone numbers, or medical info.
- [ ] Hebrew UI, RTL, at 100% zoom; capture the app window (not the whole OS desktop).
- [ ] Minimum **1280×720 px** (Capterra/GetApp requirement); PNG or JPG; keep each file < ~2 MB.
- [ ] No version number visible anywhere in frame (per §5).
- [ ] Consistent light theme and window size across all shots.

**Shots to capture (priority order):**
1. [ ] **Dashboard / home** — the at-a-glance overview after login.
2. [ ] **Patient card (כרטיס מטופל)** — allergies, ongoing treatment, previous-visit summary.
3. [ ] **Appointment calendar (יומן תורים)** — weekly view showing drag-to-reschedule.
4. [ ] **Visit documentation (תיעוד ביקורים)** — SOAP fields filled with demo content.
5. [ ] **Document archive (ארכיון מסמכים)** — list + in-document search.
6. [ ] **Encrypted backup (גיבוי מוצפן)** — backup/restore screen (password field masked).
7. [ ] **Reports & graphs (דוחות ומעקב)** — visits-per-month / hours graph *(advanced plan)*.
8. [ ] **Receipts & invoices (קבלות וחשבוניות)** *(advanced plan)*.
9. [ ] **Professional opinion editor (חוות דעת)** — template + signature + PDF export *(advanced)*.
10. [ ] **Auto-lock / PIN screen (נעילה אוטומטית)** — reinforces the privacy story.

**Also prepare (asset checklist):**
- [ ] **Logo** — "The Current" mark, brand color **#2756A6**. Square PNG with transparent
      background, at least 512×512 (G2/Capterra want a clean square). Also export a
      horizontal lockup for headers.
- [ ] **Cover / hero image** (optional, ~1200×628) for social/OG-style directory banners.
- [ ] **Short demo video** (optional but boosts ranking) — 60–120s, no audio narration needed;
      screen capture of offline workflow. Host on YouTube for Capterra/G2 embed.

## 5. Target directory list — with submission steps & review timelines

Submit in this order (fastest-value first). Track status in a simple sheet:
`Directory | Submitted date | Login/email | Status | Live URL`.

### Group A — Israeli business / local directories

#### A1. Dapei Zahav (דפי זהב) — `d.co.il`
Israel's dominant business directory; strong local-SEO signal.
- **Steps:**
  1. Go to d.co.il → "הוספת עסק / פרסום עסק" (add/advertise a business).
  2. Choose a business category — best fit: **"תוכנה / מחשבים ותוכנה"** (software) or
     **"שירותים לעסקים"**; add sub-tag for medical/clinic software in the free-text.
  3. Fill: business name **ClinicFlow**, site https://www.clinic-flow.co.il, email, phone,
     the **Hebrew SHORT** description (§1a), and logo.
  4. Submit and verify ownership (phone/email callback).
- **Note:** Dapei Zahav is largely a **paid** placement; a basic free listing exists but is
  down-ranked. Expect a sales call.
- **Review timeline:** basic listing live in ~**2–5 business days** after verification;
  paid packages activate faster once billed.

#### A2. B144 — `b144.co.il`
Second major Israeli directory (Bezeq).
- **Steps:** "הוספת עסק" → category **תוכנה/הייטק** → paste Hebrew SHORT (§1a) + logo + site
  + contact → verify by phone.
- **Review timeline:** ~**2–5 business days**; also has paid upsells.

#### A3. Zap Group — `zap.co.il`
Israeli comparison/marketplace; good for a services/software vendor page.
- **Steps:** vendor onboarding via "בעלי עסקים / הוספת עסק"; category software/services;
  Hebrew SHORT + logo + link.
- **Review timeline:** ~**3–7 business days**; mostly a paid/commercial model.

#### A4. Easy — `easy.co.il`
General Israeli directory with decent domain authority for backlinks.
- **Steps:** add business → software category → Hebrew SHORT (§1a) → verify.
- **Review timeline:** ~**2–5 business days**.

#### A5. Dun's Guide / D&B Israel — `dundun.co.il`
B2B vendor directory; credibility signal for selling software to clinics.
- **Steps:** request a company listing (B2B software vendor); provide company details + the
  Hebrew LONG (§1b) as the profile body.
- **Review timeline:** ~**1–2 weeks** (editorial/B2B vetting).

#### A6. Startup Nation Finder — `finder.startupnationcentral.org`
Curated database of Israeli tech companies. Free, high-authority backlink + credibility.
- **Steps:** "Add your company" → create company profile → sector **HealthTech / Digital
  Health**, sub-tag practice management → paste **English SHORT + LONG** (§2) → add logo,
  founding info, website → submit for editorial approval.
- **Review timeline:** ~**1–3 weeks** (manual editorial review).

> ⚠️ **Do NOT list ClinicFlow on patient-facing doctor directories** (e.g., doctor-finder
> sites, "find a clinic" listings). ClinicFlow is **software for clinicians**, not a clinic or
> a practitioner — a patient directory is the wrong audience and misrepresents the product.

#### B1. Capterra — `capterra.com`
- **Steps:**
  1. Go to Capterra → "List Your Software" / "Vendors" → create a Gartner Digital Markets
     vendor account (business email).
  2. Create the product profile: name **ClinicFlow**, website, **English SHORT + LONG** (§2),
     feature bullets (§3), pricing (**one-time, from ₪899**), screenshots, logo.
  3. **Category picks (exact):**
     - Primary: **Medical Practice Management Software**
     - Secondary: **Patient Management Software** and **Medical Scheduling Software**
     - Vertical add-ons as relevant to target professions: **Mental Health Software**,
       **Physical Therapy Software**, **Chiropractic Software**
     - EMR-adjacent: **Electronic Medical Records (EMR) Software** — list only as a
       *secondary* category, and lead with Practice Management. ⚠️ ClinicFlow is
       **EMR-adjacent, not a full EMR** (no e-prescribing, no lab/health-fund integration),
       so do not position it primarily as an EMR or buyers will expect missing capabilities.
  4. Set deployment = **Installed – On-Premise (Windows)**; pricing model = **One-time**.
  5. Submit for review.
- **Review timeline:** ~**5–10 business days** for the free listing to go live (Gartner
  moderation).

#### B2. GetApp — `getapp.com`
Same Gartner pipeline as Capterra; often auto-syncs from the shared profile.
- **Steps:** confirm the product appears from the shared Gartner profile; otherwise submit via
  "List your app." Use the **same categories** as Capterra (Medical Practice Management primary).
- **Review timeline:** ~**5–10 business days** (frequently mirrors Capterra timing).

#### B3. Software Advice — `softwareadvice.com`
Third Gartner property; benefits from the same submission.
- **Steps:** verify auto-population from the Gartner profile; category **Medical Practice
  Management**.
- **Review timeline:** ~**5–10 business days**.

#### B4. G2 — `g2.com`
Largest independent software-review platform; separate from Gartner.
- **Steps:**
  1. "Add your product" / claim profile → create a seller account (business email).
  2. Fill product details: **English SHORT + LONG** (§2), features (§3), pricing
     (**one-time, from ₪899**), screenshots, logo (square).
  3. **Category picks (exact):**
     - Primary: **Medical Practice Management Software**
     - Secondary: **Patient Scheduling Software** and **Mental Health Practice Management

     - EMR-adjacent: **EHR Software** — secondary only, same EMR caution as above.
  4. Set deployment = **On-Premise**; languages = **Hebrew**.
  5. Submit for moderation.
- **Review timeline:** ~**3 business days to ~2 weeks** for profile approval. Reviews accrue

- **SaaSHub** (`saashub.com`) — alternatives directory. Review ~a few days.
